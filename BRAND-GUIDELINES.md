| Small | 12px | Close relationships, compact layouts |

## Design Principles

### 1. Clean & Focused

Use white backgrounds with generous spacing. Prioritize one primary action per screen section. Remove visual clutter to help users focus on what matters.

### 2. Tactile Feedback

Provide pressed states on all interactive elements. Use smooth transitions with scale, opacity, and color changes. Make the interface feel responsive and alive.

### 3. Data-Driven

Present clear metrics and visualizations. Make numbers prominent and easy to scan. Use charts that are simple to understand at a glance.

### 4. Progressive Disclosure

Guide users through empty states with onboarding prompts. Hide secondary actions in contextual menus. Reveal complexity gradually as users need it.

### 5. Consistency

Apply uniform border radius (12-16px throughout). Use consistent shadows across similar components. Create predictable interaction patterns users can rely on.

## Accessibility

- Maintain WCAG AA contrast ratios minimum (4.5:1 for text)
- All interactive elements minimum 44x44px touch target
- Provide clear focus states for keyboard navigation
- Use semantic HTML/native components when possible
- Test with screen readers for critical flows
